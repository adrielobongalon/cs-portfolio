HTML

HTML is like a skeleton. It forms the basic structure of the webpage and tells the computer where content goes, like which paragraph goes where and and in what order to put them. HTML stands for HyperText Markup Language, and it was invented in 1980 by a physicist named Tim Berners-Lee. HTML is used by all websites, including this one. It supports the embedding of text, images, and other content commonly found on websites. The most recent version of HTML (as of writing) is HTML5, a revision of previous versions. It was released in October of 2014. Usually, the web browser doesn't display the actual HTML document, but rather, it displays the content that the document tells it to show.

The language of HTML consists of text and/or information surrounded by HTML tags (or just "tags", for short), which look like letters or words enclosed between angled brackets. The picture to the upper left shows an example of a tag, the <html> tag. This tag tells the web browser, "The HTML document begins here." </html> is what the corresponding closing tag would look like. Notice how the only difference between the opening and closing tags is the forward slash that precedes the tag. A paragraph would look something like this: <p>[insert words here]</p>.

Here's a list of some common HTML tags.

Common HTML Tags
tag stands for usage
<html> hypertext markup language indicates the start and end of the HTML document
<body> body body of the page, most of the webpage content goes here
<p> paragraph used for most text and paragraphs
<h1> heading, size 1 used for headings, size can be changed to whole numbers 1-6
<img> image used to insert images
<table> table used to create tables*
<div> & <span> division & span used to group content, div and span have some technical differences
*creating tables actually involves more tags than just <table>



If you want, press Ctrl (⌘ on a Mac) + U to open up the HTML document for this page in a new tab. If you do, you'll notice that most of the words you actually see on this page fall between <p> tags, which are for paragraphs. (Perhaps you might have figured out the other tags used to create tables?) You might also notice that a lot of the tags don't just "stand by themselves", so to speak. That is, you don't just see <p>, for example, by itself. They often have a bunch of words and data between the tag name ("p" in this case) and the closing angle bracket. These are called "HTML attributes". Attributes tell the web browser different information about the tag and have various purposes.

Here's a list of some common HTML attributes.

Common HTML Attributes
attribute stands for usage
class class puts stuff within a class (allows it to me modified with CSS)
id identification like a class, but meant for only one thing (the difference between class and id will be discussed later)
href hypertext reference used to reference other files or urls; tells the browser which page a link should direct to, what stylesheet to reference, and more
src source used within <img> tags to specify where to get the image from
alt alternative text used within <img> tags to specify the alternative text; in case the image is unable to show up (for whatever reason), alternative text will display
style style used to modify something; this attribute is not used too often, since CSS can completely replace it

CSS

HTML is nice for making webpages, but the webpages it can create by itself don't look too good. CSS is what allows webpages to look nice. With CSS, I can change fonts, use italics, place underlines, and create highlights with rounded corners and dark blue borders like this! (which is kind a ugly, but it shows what CSS can do) Did you notice how I styled the HTML tags like this earlier? That was CSS. There's a whole lot you can do with CSS. Some stuff is really simple, like changing the font or the size of text. Some is more complicated, like the rainbow text. Click here to see this webpage without CSS styling. You'd be surprised how much different it looks. Click here to see the CSS stylesheet for this entire website. (Both links open in new tabs.)

Especially if you looked at this page without CSS, you probably have a good grasp on how important it is. So now let's get into what CSS actually is. CSS stands for "Cascading Style Sheets". It's used by most websites, and it makes everything look pretty (or terrible if the person doesn't know how to use CSS effectively). CSS was first proposed by Håkon Wium Lie in October of 1994. He worked with Tim Berners-Lee, and at the time, several other styling languages were being proposed.

The language of CSS consists of a selector followed by opening and closing curly brackets. Within those brackets are the properties for each selector. Selectors are often html tags, ids, or classes. For example, selecting p in CSS (you don't need the angled brackets in CSS) would give all the <p> tags whatever stylings you declare for it. Setting classes and ids are especially useful in HTML because normally, you only want to style some of the tags, not all of them, and from there, you can style your selected content with CSS.

Here's a list of some common CSS properties.

Common CSS Properties
property usage
font-family changes the font
font-size changes the font
color changes the colour of a something; make sure to spell it the American way, or else it won't work. That's one of the mistakes I make rather often
background-color similar to the color property, changes the background colour of certain things
text-align changes the alignment of text
width changes the width of something
height changes the height of something
margin adjusts the margins
padding adjusts the padding
border adds a border

Documentation

Keep in mind that the HTML tags, HTML attributes, and CSS properties I listed above are not the only ones in existence. There are so many more. SO MANY MOAR! It's practically impossible to memorise all of them, and in reality, most people don't. We just remember the ones we use most often, and that depends on the person. So what do you do if you need something that you don't remember? You check the documentation. Many people have used HTML and CSS (and other programming languages), and as a result, people have created useful documents to made it easier for other people to learn. The one I personally use most often is W3Schools, which has a lot of infomation on HTML and CSS by itself. For everything else it can't answer, there are plety more documentations out there on the Internet. A big part about learning to code well is learning to use documentation effectively.
