Vindinium

Vindinium! It totally sounds like a metallic element. But it's actually a game. It's lots of fun, and it's good for learning how to code. At my school, I'm taking a computer science course as an elective, and one of our projects is to make our own bot for the game. And here, I'm going to talk about it.

The Game

a screenshot of the game (I'm not playing in this one)

So let's go over the rules and stuff. Vindinium is a turn based game. You get one player/character thingy who starts out with 100HP. And each turn, you can move one space (or stay in place). Depending on what you walk into, your guy will do different things.

The goal is to have the most amount of gold by the end of the game, which, in my experience, usually consists of 1200 turns. You can get gold by going into the mines. Problem is, they're guarded by goblins, so each time you do, you'll have to fight them. But if you survive, then the mine belongs to you, and you get 1 gold per mine you own at the end of each turn. If you step into your own mine, nothing happens. But if you step into an unowned mine, or one that belongs to someone else, then you lose 20HP, but gain control of the mine.

So obviously, you eventually gotta get your health back up, and you can do that by going into the taverns, which I call pubs, because, well let's face it. Pubs just sounds better than taverns. Anyways, at the pubs, you pay 2 gold to get 50HP.

If you try to step out of the map, or into a tree or wall, nothing happens. If your bot takes too long to figure out what to do (1 second), then your bot, he stays in place until the end of the game (but can still win if you have the most gold by the end). So make sure your bot works, or else it'll stay stuck in I-don't-know-what-to-do-mode. If you step into another player, you'll attack them, and they'll attack you. So you both lose 20HP. If you kill another player, you gain control of all their mines.

Don't worry. It's not that complicated. All this is is a fail-safe. Basically, I assign my bot both a task (what I want it to do each turn) and a myDir (direction to go in). If either I ask it not to go anywhere (myDir === "none") or if some other part of my code doesn't run properly (myDir === undefined), then my bot will choose a random direction and go that way. But most of the time, it will run the last three lines, which basically tell my bot to go whatever way I tell it to (which gets defined elsewhere in the code). All the mumbo jumbo in the middle just tries to figure out what the problem is and sends it to the "console log", which is basically a debugging tool that displays a message that you tell it to send when something in your code happens.

function rwNotCamping() {                                                       // "rw" stands for "richest weakling"
    if (richestWeakling === "none") {
        return false;
    }
    else {
        var nearestRWpub = bot.taverns[0];
        for (i = 0; i < bot.taverns.length; i++) {
            if (bot.findDistance([richestWeakling.pos.x, richestWeakling.pos.y], nearestRWpub) > bot.findDistance([richestWeakling.pos.x, richestWeakling.pos.y], bot.taverns[i])) {
                nearestPub = bot.taverns[i];
            }
        }
        if (bot.findDistance([richestWeakling.pos.x, richestWeakling.pos.y], nearestRWpub) <= 4) {
            return false;                                                       // prevents me from attacking what i call "pub campers"
        }
        else if (bot.findDistance([richestWeakling.pos.x, richestWeakling.pos.y], nearestRWpub) <= bot.findDistance(myPos, nearestRWpub)) {
            if (bot.findDistance([richestWeakling.pos.x, richestWeakling.pos.y], nearestRWpub) < 9) {
                return false;                                                   // it's not really useful to try to attack them if they're too far away (and will likely get to a
            }
            else {
                return true;
            }
        }                                                                       // pub before i can reach them)
        else {
            return true;
        }
    }
}
rwNotCamping();

It checks if the richest weakling is within 3 spaces of a pub. If it is, my bot regards it as "camping" (even if it's just getting health, I decided not to attack them because they'll heal to quickly). For everyone else, I check first where the closet pub they can reach is. Then I check if they're closer to it than I am. If I'm closer, I attack them, because it's very likely I will be able to intercept them before they can get there. If they're closer, though, then I check to make sure they're far enough from the pub. Otherwise, my bot doesn't attack them.

What I Learned

If there's one thing I can say I understand better because of this, it's arrays. There are just so many array items in Vindinium that I was pretty much forced to use them. Even the map itself is an array of arrays. And finding data about my enemies often involved me looking at arrays within arrays. But it was definitely worth learning.

I learned about array indices, how to create arrays, and how to add items into arrays. Interestingly, from my previous work from the space shooter game, I knew how to get items out of arrays (but I didn't need to use it here), but I wasn't particularly good at creating the arrays themselves. So yeah. This project basically taught me that. Yay!
